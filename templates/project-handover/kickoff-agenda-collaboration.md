## Адженда

### 1. Итоги первых двух встреч (10 мин)

- Краткая сводка: что услышал, как понял проект
- Первичные наблюдения (факты, без оценок)
- Уточняющие вопросы по расхождениям (если есть)

### 2. Формат взаимодействия (15 мин)

**Обсудить и зафиксировать:**

- Регулярный sync: как часто, с кем, в каком формате?
  - Рекомендация: еженедельно 30 мин с руководителем и/или техлидом
- Канал коммуникации: мессенджер, email, трекер?
- Как приходить с архитектурными вопросами: формат запроса
- Как архитектор даёт обратную связь: формат, куда
- Участие в code review: когда и какие PR?
- Участие во встречах команды: нужно ли?

### 3. План аудита (10 мин)

**Области аудита (8 секций):**

1. Архитектура
2. Код
3. Тесты
4. Observability
5. Deployment
6. Документация
7. Безопасность
8. Performance

**Обсудить:**
- Сроки: 2-4 недели
- Что понадобится от команды (время, доступы, ответы на вопросы)
- Формат результата: отчёт + план улучшений

### 4. Границы и ожидания (5 мин)

**Что архитектор-куратор делает:**
- Аудит и рекомендации
- Помощь в архитектурных решениях
- Ревью критичных PR и design documents

**Что архитектор-куратор НЕ делает:**
- Микроменеджмент
- Ревью каждого PR
- Принятие решений за команду

**Обратная связь в обе стороны:** если что-то не работает - говорим сразу

### 5. Следующие шаги (5 мин)

- [ ] Доступы получены (или запрос отправлен)
- [ ] Регулярный sync добавлен в календарь
- [ ] Начало аудита: дата ____
- [ ] Ожидаемая дата завершения аудита: ____
- [ ] Контактное лицо для оперативных вопросов: ____
