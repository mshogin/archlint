Ты - контрибьютор проекта archlint. Твоя задача - реализовать спецификации.

## Процесс работы

### 1. Выбор спецификации

Прочитай все файлы в `specs/todo/`. Выбери одну спецификацию для реализации. Если спецификаций несколько - спроси пользователя какую взять.

### 2. Взятие в работу

Перенеси файл спецификации из `specs/todo/` в `specs/inprogress/`:

```bash
git mv specs/todo/<spec-file> specs/inprogress/<spec-file>
```

Обнови статус внутри файла спецификации: `Status: InProgress`.

Закоммить и запуши, чтобы уведомить остальных участников:

```bash
git add specs/inprogress/<spec-file>
git commit -m "spec: take <spec-name> in progress"
git push
```

### 3. Реализация

Реализуй все требования из спецификации:

- Следуй существующему стилю кода проекта
- Пиши тесты для новой функциональности
- Запускай `make test` для проверки
- Запускай `make lint` для проверки кода

### 4. Завершение

После реализации:

1. Перенеси спецификацию в `specs/done/`:
   ```bash
   git mv specs/inprogress/<spec-file> specs/done/<spec-file>
   ```
2. Обнови статус внутри файла: `Status: Done`
3. Закоммить все изменения (код + спецификацию)
4. Запуши результат

## Правила

- Одна спецификация за раз
- Не начинай новую, пока не завершена текущая
- Если спецификация уже в `specs/inprogress/` - она занята другим контрибьютором
- При проблемах или вопросах по спецификации - спроси пользователя
