# Таксономия валидаторов archlint

Классификация ~230 валидаторов по двум осям:
- **Семейство**: Structure (статический граф) / Behavior (графы вызовов)
- **Ценность**: что защищает и что подсвечивает каждая группа

---

## Семейство: STRUCTURE (78 production + 151 research)

Работают со статическим графом зависимостей (AST-анализ).

---

### Группа 1: ЦЕЛОСТНОСТЬ ГРАФА (Structural Integrity)

**Ценность - защищает:** граф зависимостей остается управляемым, без циклов и тупиков.
**Ценность - подсвечивает:** момент, когда архитектура начинает деградировать в "клубок спагетти".

| Валидатор                     | Что проверяет                                             | Статус  |
|-------------------------------|-----------------------------------------------------------|---------|
| dag_check                     | Граф ациклический (нет circular dependencies)             | FAILED  |
| circular_dependency_depth     | Размер циклов <= 3                                        | ERROR   |
| strongly_connected_components | Взаимные зависимости (группы, которые нельзя разделить)   | WARNING |
| orphan_nodes                  | Изолированные компоненты (мертвый код?)                   | WARNING |
| graph_depth                   | Глубина графа <= 10 уровней                               | WARNING |
| graph_diameter                | Диаметр графа <= 10 (макс. расстояние между компонентами) | WARNING |

**Принципы:** DAG-property, Acyclic Dependencies Principle (ADP).

**Когда срабатывает - значит:** появилась циклическая зависимость, или граф стал слишком глубоким. Рефакторинг нужен СЕЙЧАС, пока не стало хуже.

---

### Группа 2: SOLID

**Ценность - защищает:** код легко расширяется, заменяется, тестируется. Каждый модуль делает одно дело.
**Ценность - подсвечивает:** нарушения базовых принципов, которые ведут к хрупкому, жесткому коду.

#### S - Single Responsibility

| Валидатор             | Что проверяет                            | Статус  |
|-----------------------|------------------------------------------|---------|
| single_responsibility | Модуль зависит от <= 3 доменов           | ERROR   |
| divergent_change      | Компонент меняется по <= 3 причинам      | WARNING |
| god_class             | Класс: <= 20 методов, <= 15 зависимостей | WARNING |

**Когда срабатывает:** модуль делает слишком много. Нужна декомпозиция.

#### O - Open/Closed

| Валидатор              | Что проверяет                                        | Статус  |
|------------------------|------------------------------------------------------|---------|
| open_closed            | Соотношение абстракций >= 0.2                        | ERROR   |
| speculative_generality | Абстракции имеют > 1 использования (не "на будущее") | WARNING |

**Когда срабатывает:** код нельзя расширить без изменения (мало абстракций), или наоборот - абстракции созданы впрок.

#### L - Liskov Substitution

| Валидатор           | Что проверяет                                   | Статус |
|---------------------|-------------------------------------------------|--------|
| liskov_substitution | Реализации не добавляют > 3 лишних зависимостей | ERROR  |

**Когда срабатывает:** реализация интерфейса тянет за собой неожиданные зависимости. Подменить её нельзя без побочных эффектов.

#### I - Interface Segregation

| Валидатор             | Что проверяет          | Статус  |
|-----------------------|------------------------|---------|
| interface_segregation | Интерфейс <= 5 методов | WARNING |

**Когда срабатывает:** God-интерфейс. Клиенты вынуждены зависеть от методов, которые не используют.

#### D - Dependency Inversion

| Валидатор                   | Что проверяет                                 | Статус    |
|-----------------------------|-----------------------------------------------|-----------|------------|---------|
| dependency_inversion        | Зависимости на абстракции >= 30%              | ERROR     |
| instability                 | Зависимости идут от нестабильных к стабильным | WARNING   |
| distance_from_main_sequence | D =                                           | A + I - 1 | близко к 0 | WARNING |
| abstractness                | Уровень абстрактности 0.1-0.8                 | WARNING   |

**Когда срабатывает:** высокоуровневый код зависит от деталей. Изменение инфраструктуры ломает бизнес-логику.

### Группа 3: CLEAN ARCHITECTURE

**Ценность - защищает:** бизнес-логика изолирована от инфраструктуры. Можно заменить БД, HTTP-фреймворк, внешний API без переписывания домена.
**Ценность - подсвечивает:** "протечки" между слоями, когда инфраструктура проникает в домен.

| Валидатор               | Что проверяет                                  | Статус  |
|-------------------------|------------------------------------------------|---------|
| layer_violations        | Зависимости идут только вниз по слоям          | WARNING |
| forbidden_dependencies  | handler->repository, api->storage запрещены    | FAILED  |
| inward_dependencies     | Все зависимости направлены внутрь (к domain)   | WARNING |
| domain_isolation        | Domain не зависит от infrastructure            | WARNING |
| ports_adapters          | Порты в domain, адаптеры в infrastructure      | WARNING |
| use_case_purity         | Use cases без инфраструктурного кода           | WARNING |
| dto_boundaries          | На границах - DTO, не domain entities          | WARNING |
| bounded_context_leakage | Контексты общаются через DTO/events, не типами | WARNING |
| service_autonomy        | Сервис <= 3 синхронных зависимостей            | WARNING |

**Принципы:** Dependency Rule, Ports & Adapters, Bounded Context.

**Когда срабатывает:** домен "заражен" инфраструктурой. Смена БД потребует переписывания бизнес-логики.

---

### Группа 4: DESIGN SMELLS (Антипаттерны)

**Ценность - защищает:** от накопления технического долга, который замедляет разработку.
**Ценность - подсвечивает:** конкретные code smells с конкретными решениями.

| Валидатор        | Что проверяет                              | Решение                        |
|------------------|--------------------------------------------|--------------------------------|
| god_class        | > 20 методов или > 15 зависимостей         | Декомпозиция на меньшие классы |
| feature_envy     | Метод использует чужие данные больше своих | Перенести метод к данным       |
| shotgun_surgery  | > 10 компонентов зависят от одного         | Выделить стабильный интерфейс  |
| lazy_class       | < 2 методов, < 1 зависимости               | Удалить или объединить         |
| middle_man       | Делегирует без добавления логики           | Убрать посредника              |
| data_clumps      | Компоненты всегда вместе                   | Выделить в отдельный модуль    |
| deprecated_usage | Использование deprecated компонентов       | Мигрировать на новые версии    |

---

### Группа 5: УСТОЙЧИВОСТЬ К ИЗМЕНЕНИЯМ (Change Resilience)

**Ценность - защищает:** от каскадных сбоев и неожиданного "радиуса поражения" при изменениях.
**Ценность - подсвечивает:** компоненты, изменение которых затронет пол-системы.

| Валидатор            | Что проверяет                         | Статус  |
|----------------------|---------------------------------------|---------|
| change_propagation   | Ripple effect <= 20 компонентов       | WARNING |
| blast_radius         | Радиус поражения при сбое <= 30%      | WARNING |
| hotspot_detection    | Компоненты с degree > 10              | WARNING |
| stability_violations | Нарушения стабильности в зависимостях | WARNING |
| component_complexity | Сложность компонента <= 50            | WARNING |
| coupling             | Ca <= 10, Ce <= 10                    | WARNING |

**Метрики усиления (из research):**

| Валидатор | Что добавляет |
|-----------|---------------|
| betweenness_centrality | Узкие места (bottlenecks) - если упадет, все встанет |
| pagerank | Критические компоненты по важности |
| articulation_points | Точки разрыва - удалишь компонент, граф распадется |
| bridge_edges | Критические связи - удалишь связь, граф распадется |
| eigenvector_centrality | Влиятельные узлы (влияют на влиятельных) |
| hub_nodes | God Objects по степени связности |

**Когда срабатывает:** "я изменил одну строку, а сломалось 15 сервисов". Нужна инкапсуляция.

---

### Группа 6: РАВНОМЕРНОСТЬ И БАЛАНС (Distribution & Balance)

**Ценность - защищает:** от перекосов, когда 1 модуль тянет всю нагрузку, а остальные простаивают.
**Ценность - подсвечивает:** неравномерное распределение ответственности и зависимостей.

| Валидатор | Что проверяет | Статус |
|-----------|---------------|--------|
| modularity | Компоненты хорошо изолированы (>= 0.3) | WARNING |
| edge_density | Плотность графа 0.01-0.3 | WARNING |
| gini_coefficient | Неравенство зависимостей <= 0.6 | WARNING |
| dependency_entropy | Энтропия зависимостей >= 2.0 | WARNING |
| degree_distribution | Распределение степеней | INFO |
| zscore_outliers | Статистические выбросы (z > 3.0) | WARNING |
| clustering_coefficient | Кластеризация >= 0.1 | WARNING |
| algebraic_connectivity | Связность >= 0.1 (граф не распадается) | WARNING |
| cohesion_lcom4 | Внутренняя когезия (LCOM4 <= 1) | WARNING |

**Метрики усиления (из research):**

| Валидатор | Что добавляет |
|-----------|---------------|
| spectral_radius | Спектральная характеристика связности |
| spectral_gap | Разрыв между собственными значениями - насколько "ровный" граф |
| avg_path_length | Средняя "расстояние" между компонентами |
| closeness_centrality | Насколько "центральны" ключевые узлы |
| k_core_decomposition | Ядро графа - самая связная часть |
| graph_cliques | Плотные подгруппы (взаимно зависимые) |

### Группа 7: БЕЗОПАСНОСТЬ И НАБЛЮДАЕМОСТЬ (Security & Observability)

**Ценность - защищает:** от уязвимостей на архитектурном уровне и от "слепых зон" в мониторинге.
**Ценность - подсвечивает:** дыры в безопасности и места без логирования/метрик.

| Валидатор | Что проверяет | Статус |
|-----------|---------------|--------|
| auth_boundary | Аутентификация на границе, не внутри | WARNING |
| sensitive_data_flow | Чувствительные данные не утекают в логи/кеш | WARNING |
| input_validation_layer | Валидация на входе системы | INFO |
| logging_coverage | Критические компоненты логируют | WARNING |
| metrics_exposure | Метрики для мониторинга | WARNING |
| health_check_presence | Health check endpoints есть | WARNING |
| api_consistency | API консистентность | WARNING |

---

### Группа 8: ТЕСТИРУЕМОСТЬ (Testability)

**Ценность - защищает:** код можно протестировать без боли.
**Ценность - подсвечивает:** компоненты, которые невозможно замокать или изолировать.

| Валидатор | Что проверяет | Статус |
|-----------|---------------|--------|
| mockability | Зависимости через интерфейсы >= 30% | WARNING |
| test_isolation | Тестовые компоненты изолированы от prod | WARNING |
| test_coverage_structure | Структурное покрытие >= 50% | WARNING |

### Группа 9: RESEARCH - Математический анализ архитектуры (142 валидатора)

**Статус:** Все INFO-метрики. Не проверяют принципы напрямую, но дают глубокую диагностику.

15 файлов, 15 математических доменов. Каждый домен отвечает на конкретный архитектурный вопрос.

---

#### 9.1 Базовая топология (5 валидаторов) - `topology_metrics.py`

**Архитектурный вопрос:** Какова "форма" системы? Сколько дыр (циклов), насколько она сложна?

| Валидатор | Что считает | Что значит для архитектора |
|-----------|-------------|---------------------------|
| betti_numbers | Числа Бетти (компоненты, циклы, полости) | b1 = кол-во независимых циклов = проблемных зависимостей |
| euler_characteristic | X = V - E + C | Общая "сложность формы" - чем ниже, тем запутаннее |
| simplicial_complexity | Макс. размерность клик | Насколько плотные кластеры взаимозависимостей |
| topological_persistence | Хрупкость при удалении ребер | Насколько устойчива структура при удалении одной связи |
| homological_density | Отношение циклов к максимуму | Насыщенность циклическими зависимостями |

**Ценность:** HIGH. Betti numbers -> прямой аналог cyclomatic complexity для графа зависимостей.
**Кандидат на WARNING:** `betti_numbers` (b1 > порог = слишком много циклов).

#### 9.2 Продвинутая топология (24 валидатора) - `advanced_topology_metrics.py`

**Архитектурный вопрос:** Какие структурные паттерны устойчивы, а какие случайны? Где "точки напряжения"?

| Валидатор | Что значит для архитектора |
|-----------|---------------------------|
| persistent_homology | Какие зависимости "живут" на всех масштабах (устойчивые паттерны) |
| persistence_diagram | Визуализация: birth-death пар топологических фич |
| bottleneck_stability | Устойчивость к пертурбациям (Вассерштейн расстояние) |
| forman_curvature | Где граф "изгибается" = точки напряжения между модулями |
| ricci_flow | Как напряжение распространяется по графу |
| sheaf_cohomology | Согласованность локального и глобального (контракты интерфейсов) |
| fundamental_group | Топологические циклы = фундаментальные зависимости |
| covering_space | Гомотопическое покрытие (декомпозиция на слои) |
| nerve_complex | Паттерн пересечения покрытий (как модули перекрываются) |
| vietoris_rips | Rips-комплекс с порогами расстояния |
| cech_complex | Чех-комплекс из шаровых покрытий |
| morse_complexity | Количество критических точек (дискретная теория Морса) |
| morse_smale | Декомпозиция Морса-Смейла |
| gradient_flow | Дискретный градиентный поток |
| hodge_decomposition | Разложение на гармоническое, точное, коточное |
| weight_filtration | Персистентность при взвешенной фильтрации |
| persistence_landscape | "Ландшафт" персистентных фич |
| alpha_complex | Делоне-подкомплекс (взвешенный) |
| discrete_exterior_calculus | Дискретные дифференциальные формы |
| spectral_sequence | Сходимость спектральной последовательности |
| path_homology | Гомология на основе путей |
| wasserstein_stability | Вассерштейн расстояние между диаграммами |
| local_consistency | Согласованность локальных окрестностей |
| homotopy_equivalence | Анализ топологического класса эквивалентности |
| clique_complex | Флаг-комплекс из пересечений клик |

**Ценность:** MODERATE. Самый большой домен, глубокий мат. аппарат. Реальная ценность в 5-6 метриках (persistent_homology, forman_curvature, sheaf_cohomology, nerve_complex, bottleneck_stability, fundamental_group). Остальные - академический фундамент.

**Кандидат на WARNING:** `forman_curvature` (выбросы кривизны = экстремальные точки напряжения).

#### 9.3 Теория информации (8 валидаторов) - `information_theory_metrics.py`

**Архитектурный вопрос:** Сколько "информационного шума" в архитектуре? Насколько предсказуемы зависимости?

| Валидатор | Что значит для архитектора |
|-----------|---------------------------|
| mutual_information | Насколько модули "знают" друг о друге (high = tight coupling) |
| conditional_entropy | Неопределенность при знании части системы |
| channel_capacity | Пропускная способность информации через компонент |
| kolmogorov_complexity | Минимальное описание архитектуры (high = сложная) |
| markov_stationary | PageRank-важность компонентов |
| absorption_probability | Вероятность "утечки" в leaf-узлы |
| mixing_time | Время достижения стационарного распределения |
| cross_entropy | Отклонение от равномерного распределения зависимостей |

**Ценность:** HIGH. mutual_information - прямая метрика coupling. kolmogorov_complexity - "сжимаемость" архитектуры.
**Кандидат на WARNING:** `mutual_information` (> порог = слишком тесная связь между пакетами).

#### 9.4 Линейная алгебра (8 валидаторов) - `linear_algebra_metrics.py`

**Архитектурный вопрос:** Насколько система чувствительна к возмущениям? Можно ли ей управлять?

| Валидатор | Что значит для архитектора |
|-----------|---------------------------|
| matrix_rank | Независимость компонентов (low rank = много дублирования) |
| condition_number | Чувствительность к изменениям (high = хрупкая система) |
| svd_analysis | Эффективная размерность (реальное число независимых модулей) |
| spectral_gap | Разрыв между собственными значениями = "ровность" графа |
| lyapunov_stability | Устойчивость системы (все eig < 0 = стабильна) |
| controllability | Управляемость системы (ранг матрицы управляемости) |
| graph_curvature | Оливье-Риччи кривизна (дискретная) |
| effective_resistance | Сопротивление графа (сила связности) |

**Ценность:** HIGH. condition_number - прямой аналог "хрупкости" системы. svd_analysis - реальная размерность.
**Кандидат на WARNING:** `condition_number` (> 100 = архитектура очень чувствительна к изменениям).

#### 9.5 Продвинутая теория графов (7 валидаторов) - `advanced_graph_metrics.py`

**Архитектурный вопрос:** Насколько архитектура "древовидна"? Можно ли её разбить на независимые группы?

| Валидатор | Что значит для архитектора |
|-----------|---------------------------|
| treewidth | Насколько "древовидная" архитектура (low = хорошо, легко декомпозировать) |
| chromatic_number | Мин. число "цветов" = независимых групп деплоя |
| dominating_set | Мин. набор компонентов для полного контроля |
| independence_number | Макс. набор компонентов без взаимных зависимостей |
| vertex_cover | Мин. набор компонентов для покрытия всех связей (= тестовое покрытие) |
| graph_density_distribution | Равномерность локальной плотности |
| graph_symmetry | Регулярность графа (автоморфизмы) |

**Ценность:** HIGH. treewidth - ключевая метрика декомпозируемости. vertex_cover - минимальный тестовый набор.
**Кандидат на WARNING:** `treewidth` (> 5 = система плохо декомпозируется).
